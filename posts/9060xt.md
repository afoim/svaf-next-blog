# 小坑

- 在 AMD Readon Software 中，建议将所有设置设为默认。对于增值设置：如 **FSR 超分辨率**，**FSR 帧生成** ，则将其关闭。因为在某些不支持这些设置的游戏中，AMD驱动仍然会向游戏声明或覆盖这些设置，可能会导致 **Low帧低** 或 **显示异常** 。推荐在游戏中设置这些选项（若目标游戏支持）

	![](/img/9060xt-index.png)

- 若启用 **FSR 帧生成** ，则可能会在游戏中遇到 **伪影** ，特别是针对高速移动的物品处理，FSR并没有DLSS那样能处理得得心应手。而且性能提升并不大，对于低显存显卡，建议 **仅启用 FSR 超分辨率**

	![](/img/9060xt-4fcd082a176b6f5f82e516a77b0a7859.png)

	![](/img/9060xt-d339d33e7a92ac8a331c535750c649b8.png)

# 测试平台

* **处理器：** 英特尔 第十三代酷睿 i5-13600KF 十四核
* **主板：** 微星 MAG B760M MORTAR MAX WIFI (MS-7E01) ( 英特尔 B760 芯片组 )
* **内存：** 32GB DDR5 6287MHz ( 16GB + 16GB )
* **显卡：** AMD Radeon RX 9060 XT ( 8GB )
* **磁盘：** Fanxiang S790E 1TB ( 1000GB )
* **显示器：** Mi Monitor [小米 XMI27B1] ( 27.2英寸 )（2560x1440 **2K**）

# 详细性能指标

> 以下内容的设置倾向为 **能用 FSR 超分辨率 就用**，若游戏不支持则**强算**。始终不使用 **FSR 帧生成** 。测试分辨率为 **2K**，具体值为上一节提到的测试平台使用的显示器

> [!tip]
> 对于低显存的显卡，在游戏中频繁更改画质设置可能会导致显存内临时存放多个不同质量的材质，导致 **爆显存** ，主观体验为：**极低的帧率，游戏画面卡住，游戏程序崩溃** 。建议在进行画质调整后，静等一段时间，等待旧的材质在显存中被清理释放，或者重启游戏后再进行下一次的画质调整。否则会一直处于：**加载新材质 --> 爆显存 --> 垃圾清理未完毕 --> 加载新材质 --> 继续爆显存** 

- **绝区零** ：画质拉满**≈130FPS**（不支持也不接受任何FSR相关设置）
- **明日方舟：终末地** ：画质拉满**≈80FPS**（不支持也不接受任何FSR相关设置。若 **强制全局启用FSR的任何设置** ，会频繁出现顿卡，导致Low帧低）
- **鸣潮：** 画质拉满**≈100FPS**，若启用FSR超分辨率，则**≈170FPS**
- **卡拉彼丘**： 最低画质靶场顶着**480FPS**跑（FPS游戏，帧数＞画质，还打不死人就肯定你的问题）
- **异环** ：平衡画质**≈100FPS**，若启用FSR超分辨率，则**≈170FPS**（该游戏在更高一档预设画质，即 **极致** 中，会**新增反射相关的设置**，且默认会分配 **SSR** 选项 ，会导致帧数断档性质的下降，同时画质也会有**质的飞跃**，若启用，只剩**60FPS**左右，为了最终体验，不再拉高画质）
